# 병합 정렬 (Merge Sort)

## 1. 개요 및 복잡도
### 시간 복잡도
최선 : $O(nlogn)$  
평균 : $O(nlogn)$  
최악 : $O(nlogn)$

### 공간 복잡도
$O(n)$ (병합 과정을 위한 추가적인 배열 공간이 필요함)

### 핵심 요약
- 분할 정복 알고리즘의 하나로, 배열을 가장 작은 단위(원소 1개)까지 반씩 쪼갠 뒤, 다시 크기 순으로 합치면 정렬하는 방식  
- 데이터의 초기 상태와 무관하게 항상 일정한 시간 복잡도를 보장하는 매우 안적적이고 효율적인 정렬 알고리즘

## 2. 동작 원리
1. 정렬되지 않은 배열을 절반으로 잘라 비슷한 크기의 두 부분 배별로 나눈다.  
2. 배열의 원소가 1개가 될 때까지 이 분할 과정을 재귀적으로 반복한다. (원소가 1개인 배열은 이미 정렬된 것으로 간주)  
3. 두 개의 분할된 배열을 순서대로 비교하며 하나의 정렬된 배열로 합친다.  
4. 병합 시, 두 부분 배열의 첫 번째 원소들을 서로 비교하여 더 작은 값을 새로운 임시 배열에 순서대로 채워 넣는다.  
5. 모든 부분 배열이 하나로 병합될 때까지 이 과정을 반복한다.

## 3. 장단점 및 특징
### 장점
- 데이터의 분포나 초기 상태에 영향을 받지 않고 항상 $O(nlogn)$의 빠른 정렬 속도를 보장한다.  
- 크기가 큰 데이터를 정렬할 때 효율적이다.  
- 연결 리스트를 정렬할 때 사용하면 데이터의 이동 없이 링크만 변경하므로 제자리 정렬(In-place sort)처럼 효율적으로 구현할 수 있다.

### 단점
- 정렬 과정에서 원본 배열과 같은 크기의 추가적인 메모리 공간 ($O(n)$)이 반드시 필요하다.  
- 배열의 크기가 클 경우, 임시 배열로 데이터를 복사하고 다시 원본 배열로 가져오는 과정에서 이동 횟수가 많아져 오버헤드가 발생할 수 있다.

### 특이사항
- 안정 정렬 : 값이 같은 원소끼리는 정렬 전의 원래 순서가 그대로 유지된다.  
- Out-of-place Sort : 제자리 정렬이 아니며, 별도의 추가 메모리 공간을 요구한다.

## 4. 구현 (C++)
[MergeSort.cpp 전체 코드 파일 확인](./MergeSort.cpp)
```cpp
#include <vector>

using namespace std;

// 두 개의 정렬된 부분 배열을 병합하는 함수
void Merge(vector<int>& arr, int left, int mid, int right) {
    int n1 = mid - left + 1;
    int n2 = right - mid;

    // 임시 배열 생성
    vector<int> L(n1), R(n2);

    // 데이터 복사
    for (int i = 0; i < n1; i++) L[i] = arr[left + i];
    for (int j = 0; j < n2; j++) R[j] = arr[mid + 1 + j];

    int i = 0, j = 0, k = left;

    // 두 부분 배열을 비교하며 원본 배열에 정렬하여 덮어쓰기
    while (i < n1 && j < n2) {
        if (L[i] <= R[j]) {
            arr[k] = L[i];
            i++;
        } else {
            arr[k] = R[j];
            j++;
        }
        k++;
    }

    // 왼쪽 배열에 남은 원소가 있다면 모두 복사
    while (i < n1) {
        arr[k] = L[i];
        i++;
        k++;
    }

    // 오른쪽 배열에 남은 원소가 있다면 모두 복사
    while (j < n2) {
        arr[k] = R[j];
        j++;
        k++;
    }
}

// 분할 정복을 수행하는 메인 정렬 함수
void MergeSort(vector<int>& arr, int left, int right) {
    if (left >= right) return; // 원소가 1개 이하면 종료

    int mid = left + (right - left) / 2; // 중간 지점 계산 (오버플로우 방지)

    MergeSort(arr, left, mid);       // 왼쪽 절반 분할
    MergeSort(arr, mid + 1, right);  // 오른쪽 절반 분할
    Merge(arr, left, mid, right);    // 분할된 두 배열 병합
}
```

# 퀵 정렬 (Quick Sort)

## 1. 개요 및 복잡도
### 시간 복잡도
최선 : $O(nlogn)$ (피벗이 배열을 항상 정확히 반으로 나누는 경우)  
평균 : $O(nlogn)$  
최악 : $O(n^2)$ (이미 정렬된 배열에서 피벗을 항상 최솟값이나 최댓값으로 잡을 때 발생)

### 공간 복잡도
평균 : $O(logn)$  
최악 : $O(n)$ (재귀 호출에 따른 스택 메모리 사용량이며, 배열 자체의 이동을 위한 추가 메모리는 거의 사용하지 않음)

### 핵심 요약
- 분할 정복 알고리즘의 하나로, 기준점인 피벗을 설정하고 피벗보다 작은 값은 왼쪽, 큰 값은 오른쪽으로 모으는 과정을 반복하여 정렬하는 방식  
- c++의 `std::sort` 등 대다수 프로그래밍 언어의 내장 정렬 라이브러리에서 기본적으로 채택할 만큼, 실무에서 가장 빠르고 범용적으로 쓰이는 정렬 알고리즘

## 2. 동작 원리
1. 배열에서 임의의 원소 하나를 고른다. 이 원소를 피벗이라 부른다. (보통 첫 번째, 중간, 또는 마지막 원소를 선택)  
2. 피벗을 기준으로 배열을 탐색하며, 피벗보다 작은 원소들은 모두 피벗의 왼쪽으로, 피벗보다 큰 원소들은 모두 피벗의 오른쪽으로 옮긴다. (이 과정이 끝나면 피벗의 최종 위치 확정됨)  
3. 피벗을 기준으로 나뉜 왼쪽 부분 배열과 오른쪽 부분 배열에 대해 각각 새로운 피벗을 설정하고 분할 과정을 재귀적으로 반복한다.  
4. 부분 배열의 크기가 1 이하가 되면 정렬이 완료된 것으로 간주하고 재귀를 종료한다.

## 3. 장단점 및 특징
### 장점
- 평균 시간 복잡도가 $O(n\log n)$인 다른 정렬 알고리즘(병합 정렬, 힙 정렬 등)과 비교했을 때도 실제 동작 속도가 압도적으로 빠르다.  
- 병합 정렬과 달리, 추가적인 배열(메모리) 공간을 생성하지 않고 기존 배열 내에서 자리를 바꾸는 방식이므로 메모리 낭비가 적다.

### 단점
- 피벗을 잘못 설정할 경우, 분할이 한쪽으로만 쏠리게 되어 최악의 시간 복잡도인 $O(n^2)$까지 성능이 급격히 저하될 수 있다.  
- 값이 같은 원소들의 원래 순서가 섞일 수 있는 불안정 정렬이다.

### 특이사항
- 불안정 정렬 : 값이 같은 데이터의 초기 순서가 정렬 후에도 유지된다는 보장이 없다.  
- In-place Sort : 제자리 정렬로, 재귀 호출 스택을 제외하면 데이터 정렬 자체를 위한 추가적인 힙 메모리 공간이 필요하지 않다.

## 4. 구현 (C++)
[QuickSort.cpp 전체 코드 파일 확인](./QuickSort.cpp)
```cpp
#include <vector>
#include <algorithm> // swap 함수 사용

using namespace std;

// 퀵 정렬 메인 함수 (중간값을 피벗으로 사용하는 방식)
void QuickSort(vector<int>& arr, int left, int right) {
    if (left >= right) return; // 원소가 1개 이하면 종료

    // 배열의 중간 위치의 값을 피벗으로 설정 (최악의 경우 방지용)
    int pivot = arr[(left + right) / 2];
    int i = left;
    int j = right;

    // 분할 (Partition) 과정
    while (i <= j) {
        // 왼쪽 포인터(i)는 피벗보다 크거나 같은 값을 찾을 때까지 이동
        while (arr[i] < pivot) i++;
        // 오른쪽 포인터(j)는 피벗보다 작거나 같은 값을 찾을 때까지 이동
        while (arr[j] > pivot) j--;

        // 두 포인터가 교차하지 않았다면 두 값을 교환하고 포인터 이동
        if (i <= j) {
            swap(arr[i], arr[j]);
            i++;
            j--;
        }
    }

    // 피벗을 기준으로 나뉜 두 부분 배열에 대해 재귀 호출
    QuickSort(arr, left, j);   // 왼쪽 부분 정렬
    QuickSort(arr, i, right);  // 오른쪽 부분 정렬
}
```
