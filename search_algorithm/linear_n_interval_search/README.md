# 순차 탐색 (Sequential Search)

## 1. 개요 및 복잡도
### 시간 복잡도
최선 : $O(1)$ (찾고자 하는 데이터가 배열의 맨 앞에 위치한 경우)  
평균 : $O(n)$ (데이터가 배열의 중간쯤에 있거나 무작위로 분포되어 있을 때 발생)  
최악 : $O(n)$ (찾고자 하는 데이터가 배열의 맨 끝에 있거나 아예 존재하지 않는 경우)  

### 공간 복잡도
$O(1)$ (별도의 추가 메모리 공간 없이 주어진 배열 내에서 탐색을 수행)  

### 핵심 요약
- 배열이나 리스트의 처음부터 끝까지 순서대로 하나씩 데이터를 비교하며 원하는 값을 찾는 방식  
- 데이터가 정렬되어 있지 않은 상태에서도 제약 없이 사용할 수 있는 가장 단순하고 직관적인 탐색 알고리즘  

## 2. 동작 원리
1. 배열의 첫 번째 인덱스(0번)부터 탐색을 시작한다.  
2. 현재 인덱스의 데이터와 찾고자 하는 대상 데이터가 일치하는지 비교한다.  
3. 일치하면 해당 데이터의 인덱스를 반환하고 탐색을 즉시 종료한다.  
4. 일치하지 않으면 다음 인덱스로 이동하여 2번 과정을 반복한다.  
5. 배열의 끝까지 탐색했음에도 대상을 찾지 못하면 탐색 실패(예 : `-1` 반환)로 처리한다.  


## 3. 장단점 및 특징
### 장점
- 데이터의 정렬 여부와 상관없이 항상 정확하게 원하는 값을 찾아낼 수 있다.  
- 알고리즘이 매우 직관적이고 구현이 단순하여 데이터 양이 적을 때 효율적으로 사용할 수 있다.  

### 단점
- 데이터의 총 개수($n$)가 많아질수록 탐색에 걸리는 시간이 선형적으로 증가하여, 대규모 데이터 처리에는 매우 비효율적이다.  

### 특이사항
- 선형 탐색 (Linear Search) : 순차 탐색을 부르는 또 다른 이름이다.  

## 4. 구현 (C++)
[SequentialSearch.cpp 전체 코드 파일 확인](./SequentialSearch.cpp)
```cpp
#include <vector>

using namespace std;

// 순차 탐색 메인 함수
// 찾고자 하는 target이 있으면 해당 인덱스를, 없으면 배열의 크기를 반환
int SequentialSearch(const vector<int>& arr, int target) {
    int n = arr.size();

    // 배열의 처음부터 끝까지 순차적으로 탐색
    for (int i = 0; i < n; i++) {
        if (arr[i] == target) {
            return i; // 일치하는 데이터를 찾으면 즉시 인덱스 반환
        }
    }

    // 배열을 끝까지 순회했음에도 찾지 못한 경우
    return arr.size(); 
}
```
