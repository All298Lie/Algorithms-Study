# 선형/구간 탐색 (Linear & Interval Search)

# 순차 탐색 (Sequential Search)

## 1. 개요 및 복잡도
### 시간 복잡도
최선 : $O(1)$ (찾고자 하는 데이터가 배열의 맨 앞에 위치한 경우)  
평균 : $O(n)$ (데이터가 배열의 중간쯤에 있거나 무작위로 분포되어 있을 때 발생)  
최악 : $O(n)$ (찾고자 하는 데이터가 배열의 맨 끝에 있거나 아예 존재하지 않는 경우)  

### 공간 복잡도
$O(1)$ (별도의 추가 메모리 공간 없이 주어진 배열 내에서 탐색을 수행)  

### 핵심 요약
- 배열이나 리스트의 처음부터 끝까지 순서대로 하나씩 데이터를 비교하며 원하는 값을 찾는 방식  
- 데이터가 정렬되어 있지 않은 상태에서도 제약 없이 사용할 수 있는 가장 단순하고 직관적인 탐색 알고리즘  

## 2. 동작 원리
1. 배열의 첫 번째 인덱스(0번)부터 탐색을 시작한다.  
2. 현재 인덱스의 데이터와 찾고자 하는 대상 데이터가 일치하는지 비교한다.  
3. 일치하면 해당 데이터의 인덱스를 반환하고 탐색을 즉시 종료한다.  
4. 일치하지 않으면 다음 인덱스로 이동하여 2번 과정을 반복한다.  
5. 배열의 끝까지 탐색했음에도 대상을 찾지 못하면 탐색 실패(예 : `-1` 반환)로 처리한다.  


## 3. 장단점 및 특징
### 장점
- 데이터의 정렬 여부와 상관없이 항상 정확하게 원하는 값을 찾아낼 수 있다.  
- 알고리즘이 매우 직관적이고 구현이 단순하여 데이터 양이 적을 때 효율적으로 사용할 수 있다.  

### 단점
- 데이터의 총 개수($n$)가 많아질수록 탐색에 걸리는 시간이 선형적으로 증가하여, 대규모 데이터 처리에는 매우 비효율적이다.  

### 특이사항
- 선형 탐색 (Linear Search) : 순차 탐색을 부르는 또 다른 이름이다.  

## 4. 구현 (C++)
[SequentialSearch.cpp 전체 코드 파일 확인](./SequentialSearch.cpp)
```cpp
#include <vector>

using namespace std;

// 순차 탐색 메인 함수
// 찾고자 하는 target이 있으면 해당 인덱스를, 없으면 배열의 크기를 반환
int SequentialSearch(const vector<int>& arr, int target) {
    int n = arr.size();

    // 배열의 처음부터 끝까지 순차적으로 탐색
    for (int i = 0; i < n; i++) {
        if (arr[i] == target) {
            return i; // 일치하는 데이터를 찾으면 즉시 인덱스 반환
        }
    }

    // 배열을 끝까지 순회했음에도 찾지 못한 경우
    return arr.size(); 
}
```

# 이진 탐색 (Binary Search)

## 1. 개요 및 복잡도
### 시간 복잡도
최선 : $O(1)$ (찾고자 하는 데이터가 배열의 정확히 한가운데에 위치한 경우) 
평균 : $O(\log n)$  
최악 : $O(\log n)$ (데이터가 배열에 존재하지 않거나 끝단에 있어, 범위를 1개가 될 때까지 계속 나누는 경우) 

### 공간 복잡도
$O(1)$ (반복문을 사용하여 구현할 경우, 별도의 추가 배열 없이 시작점, 끝점, 중간점을 나타내는 변수 메모리만 필요)  

### 핵심 요약
- 탐색 범위를 절반씩 쪼개어 나가며 원하는 데이터가 있을 만한 곳을 빠르게 좁혀가는 방식
- 순차 탐색($O(n)$)과 비교할 수 없을 정도로 압도적인 속도를 자랑하지만, 반드시 데이터가 사전에 ‘정렬되어 있어야만’ 동작하는 조건부 알고리즘  

## 2. 동작 원리
1. 탐색할 배열의 양 끝에 시작점(Left)과 끝점(right)을 설정하고, 그 중앙에 위치한 인덱스를 중간점(Mid)으로 지정한다. (소수점은 버린다.)
2. 중간점에 위치한 데이터와 찾고자 하는 데상 데이터의 크기를 비교한다.
3. 대상과 중간점 데이터가 일치하면 탐색을 즉시 종료하고 해당 인덱스를 반환한다.
4. 데상이 중간점 데이터보다 작다면, 대상은 무조건 중간점의 왼쪽에 존재하므로 탐색 범위를 왼쪽 절반으로 줄인다. (끝점 = 중간점 - 1)
5. 대상이 중간점 데이터보다 크다면, 대상은 무조건 중간점의 오른쪽에 존재하므로 탐색 범위를 오른쪽 절반으로 줄인다. (시작점 = 중간점 + 1)
6. 대상을 찾거나, 탐색 범위가 더 이상 존재하지 않을 때(시작점이 끝점보다 커지는 교차 현상 발생 시)까지 1~5번 과정을 반복한다.


## 3. 장단점 및 특징
### 장점
- 탐색을 한 번 수행할 때마다 확인해야 할 데이터의 개수가 절반씩 줄어들기 때문에, 데이터가 수억 개가 넘어가도 단 몇 십 번의 비교만으로 탐색을 완료할 수 있다.
- 순차 탐색 대비 성능이 기하급수적으로 좋으며, 구현 구조도 깔끔하다.

### 단점
- 배열이 사전에 오름차순(또는 내림차순)으로 완벽하게 정렬되어 있어야만 사용할 수 있다는 치명적인 전제 조건이 있다.
- 데이터의 삽입과 삭제가 빈번하게 일어나는 환경에서는, 매번 정렬 상태를 유지하기 위한 비용이 이진 탐색으로 얻는 이득보다 커질 수 있다.

### 특이사항
- 파라메트릭 서치 (Parametric Search) : 최적화 문제(예 : 조건을 만족하는 최댓값 구하기)를 결정 문제(예 : 이 값일 때 조건을 만족하는가? ‘예/아니오’)로 바꾸어 해결하는 기법으로, 코딩 테스트에서 이진 탐색의 원리를 응용하여 출제되는 가장 대표적인 심화 알고리즘이다.
- 오버플로우 주의 : 중간점(Mid)을 구할 때 (Left + right) / 2 방식을 사용하면 두 값의 합이 자료형의 최댓값을 초과할 위험이 있으므로, 실무에서는 Left + (Right - Left) / 2 방식을 사용하는 것이 안전하다.

## 4. 구현 (C++)
[BinarySearch.cpp 전체 코드 파일 확인](./BinarySearch.cpp)
```cpp
#include <vector>
#include <iostream>

using namespace std;

// 이진 탐색 메인 함수 (반복문 사용)
// 찾고자 하는 target이 있으면 해당 인덱스를, 없으면 -1을 반환
int BinarySearch(const vector<int>& arr, int target) {
    int left = 0;
    int right = arr.size() - 1;

    // 시작점이 끝점을 역전하기 전까지 반복
    while (left <= right) {
        // (left + right) / 2 와 같지만 오버플로우를 방지하는 안전한 수식
        int mid = left + (right - left) / 2;

        // 1. 중간점의 데이터가 타겟과 일치하는 경우
        if (arr[mid] == target) {
            return mid;
        }
        // 2. 타겟이 중간점의 데이터보다 작은 경우 (왼쪽 절반 탐색)
        else if (target < arr[mid]) {
            right = mid - 1;
        }
        // 3. 타겟이 중간점의 데이터보다 큰 경우 (오른쪽 절반 탐색)
        else {
            left = mid + 1;
        }
    }

    // 배열을 끝까지 탐색했음에도 찾지 못한 경우
    return arr.size(); 
}
```
